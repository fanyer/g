# Go Version Manager

For Shell to advance
===

以下的基准环境均为 OSX10.11.8

Golang1.4之后的版本需要一个本地存在其他版本的go环境实现自举，在1.4之前的版本需要gcc和glibc环境来编译。

因此任何使用源码更新编译的版本管理都预先需要

 1. 一个本地已有的go环境
 2. 或是配置编译go的1.4版本所需要的gcc和glibc（Mac不支持）环境。
 
 这看起来非常类似于n管理node的版本。不同的是，n使用的是直接抓取官方生成的二进制可执行文件，而GVM则是嗅探源码进行本地编译然后实现版本控制的。这种方式其实能最大程度保证对本地机器的兼容。

值得指出的是，golang官网的安装包会有godoc这一辅助文档工具。若从golang源码编译出的则仅有go、gofmt两个可执行文件。
